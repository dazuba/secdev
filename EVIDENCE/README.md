# Evidence Artifacts

This directory contains security scan results and evidence artifacts generated by CI/CD pipelines.

## Structure

### P09/ - SBOM & SCA (Supply Chain Analysis)

Generated by `.github/workflows/ci-sbom-sca.yml`

- `sbom.json` - Software Bill of Materials in CycloneDX format (generated by Syft v1.21.0)
- `sca_report.json` - Security vulnerability scan report (generated by Grype v0.85.0)
- `sca_summary.md` - Human-readable summary of vulnerabilities by severity with action plan

### P10/ - SAST & Secrets Scanning

Generated by `.github/workflows/ci-sast-secrets.yml`

- `semgrep.sarif` - SAST findings in SARIF format (generated by Semgrep v1.103.0)
- `gitleaks.json` - Secrets detection report (generated by Gitleaks v8.21.2)
- `sast_summary.md` - Human-readable summary of SAST findings and secrets with triage decisions

### Artifact Retention

- CI artifacts are retained for 90 days in GitHub Actions
- Local copies in this directory are committed for traceability
- Each artifact includes commit SHA and job URL for complete traceability

### Usage

These artifacts are used for:
- DS1 (Dependency Security) - P09 SBOM & SCA results
- DS2 (Static Analysis) - P10 SAST & Secrets findings
- Security compliance documentation
- Vulnerability tracking and waiver management (see `policy/waivers.yml`)
- Supply chain security audit trail
- Code quality and secure coding practices verification
